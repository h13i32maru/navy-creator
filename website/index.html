Overview

Goal

「ブラウザをアプリケーションプラットフォームとすること」をゴールと考えています。

しかし現状はこのゴールが受け入れられるのか？実現可能なのか？を確かめるための最小限のフレームワーク(コンセプトフレームワーク)として開発しています。

NavyJS

SPA(Single Page Application)、View、アセットなどの仕組みを提供するモバイル用のWebアプリケーションフレームワークです。

対応ブラウザ

  • Chrome for Android
  • Mobile Safari
  • Google Chrome

NavyCreator

GUIで画面レイアウトを作成したり、アセットの管理を行うための専用開発環境(簡易IDE)です。

対応OS

  • Mac OSX

NavyCreator

Feature

View

JS/HTML/CSSをパッケージングし、より高度で再利用可能な部品としてボタン、リスト、タブなどのViewを提供します。

Layout

画面上に配置するViewをGUI(NavyCreator)を使って作成することができます。 作成されたレイアウトファイル(JSON)をNavyJSが読み込むことでブラウザ上にHTMLを動的に構築し画面を描画します。 レイアウトから他のレイアウトを読み込むことことでレイアウト自体を再利用できます。

SPA

画面単位で独立した状態を保持することができるので、シングルページアプリケーションを簡単に構築することができます。 画面のライフサイクル(生成、表示、非表示、破棄)を管理する機能を提供します。

Asset

アセット(JS/レイアウト/画像)をブラウザに擬似的にインストールすることでネットワークアクセスをせずにアセットを利用することができます。 Webサーバ上で新しいアセットが公開されると差分更新することができます。

注意: 現状、画像はブラウザキャッシュを利用しているため、サーバ側での適切なレスポンスヘッダの設定が必要になります。
